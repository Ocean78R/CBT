# Политика обязательного обновления документации

## Статус
Обязательно к исполнению для всех новых доработок.

## Правило
Любая новая функция или изменение поведения бота должно сопровождаться обновлением документации в том же изменении кода.

## Что считается изменением, требующим документации
- новая функциональность,
- новый config-параметр,
- новый режим работы,
- изменение runtime-пайплайна,
- изменение risk/entry/exit/lifecycle логики,
- изменение контрактов модулей, влияющих на оператора.

## Минимальный комплект обновлений
1. Обновить соответствующий раздел `docs/user/*`.
2. Добавить запись в `docs/DOCUMENTATION_CHANGELOG_RU.md`.
3. Добавить русские комментарии рядом с новой/изменённой логикой в коде (где применимо).

## Правило в коде
В файлах с новой функциональностью рядом с изменённой логикой должны быть русские комментарии, объясняющие:
- место слоя в пайплайне,
- зависимость от ранних слоёв,
- fallback-поведение.

## Контроль соблюдения
В проекте действует автоматическая проверка `npm run test:docs`, которая валидирует:
- наличие policy-документа,
- наличие user-docs раздела,
- наличие журнала изменений документации,
- обязательную запись в changelog при изменениях кода.
