# Пользовательская документация CBT (для оператора)

## 1. Назначение бота
CBT — торговый бот для автоматизированной работы с фьючерсными инструментами по заранее заданным правилам риск-контроля, входа, сопровождения и закрытия позиции.

Документация ниже ориентирована на оператора (не на разработчика):
- как безопасно запустить,
- как читать состояние,
- какие параметры допустимо менять,
- как действовать при сбоях.

## 2. Быстрый старт
1. Настройте `dist/_config/config.json` под вашу биржу/аккаунт.
2. Проверьте значения risk-параметров и режимов запуска.
3. Выполните запуск в целевом окружении через штатный сценарий проекта.
4. Убедитесь, что в логах нет hard-veto/ошибок коннектора на старте.

Подробный порядок: [Запуск и режимы](./RUN_AND_MODES_RU.md).

## 4. Границы ответственности оператора
Оператор:
- меняет только параметры, явно помеченные как операторские,
- не отключает защитные контуры без отдельного согласования,
- фиксирует каждую функциональную доработку в changelog документации.

Если вы не уверены в параметре — используйте консервативное значение и сверяйтесь с разделом «Опасные параметры» в [CONFIG_GUIDE_RU.md](./CONFIG_GUIDE_RU.md).
